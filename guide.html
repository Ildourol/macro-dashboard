Dashboard Index & Operations Guide

A top-down operating map for reading the terminal like an institutional futures desk, plus the refresh commands and skills workflow that keep it stable.

1. Executive Workflow

Step 1
Regime
Start with Rates/SOFR, volatility, gamma, and the calendar. These define how much risk the market can absorb.
Step 2
Sponsorship
Use Volume/OI, COT, options skew, FX, and sector flow to see whether price action has durable participation.
Step 3
Execution
Move to Trade Setups and Risk Lab only after the regime and sponsorship checks agree with the trade direction.
Desk rule: No futures trade should be justified by a single page. A setup becomes actionable only when regime, flow, positioning, catalyst risk, and invalidation align.

3. Institutional Interpretation Rules

Continuation
Continuation requires price strength, stable or rising OI, supportive rates, and no hostile options skew. One missing pillar lowers size.
Reversal
Reversal risk rises when price reaches resistance while OI falls, put skew rises, gamma support is stale, or COT crowding is extreme.
Liquidity Trap
High volume after a sharp move is not bullish by itself. If OI does not rebuild, the move can be distribution, short covering, or forced flow.
Capital Preservation
When signals conflict, reduce notional first. The dashboard is designed to protect capital before maximizing upside.

4. Command Map project shortcuts

update
Refresh ritual
Downloads the known source pack, shows status, verifies HTML structure, then prints the next analysis step.
download
Source pack
Uses the latest refresh manifest and BITS transfer method to refresh PDFs into the local archive.
dash-status
Freshness
Reports latest refresh/extraction manifests, local PDF count, and whether the skills pack is present.
dash-verify
Structure
Checks section balance, required files, and whether every HTML page carries the combined Index navigation link.
How to activate aliases: in PowerShell from the dashboard folder, run Set-ExecutionPolicy -Scope Process Bypass, then . .\tools\load-dashboard-aliases.ps1. After that, the short commands above work in the same session. The Windows-friendly launchers are update.cmd, download.cmd, verify.cmd, and serve.cmd.

7. Refresh Boundaries execution caveats

Automated
Known PDF source downloads, local manifest creation, status checks, page-structure checks, and local serving.
Manual / Agent Review
Source interpretation, chart/table edits, gamma browser capture, blocked JavaScript pages, and final trade-date labeling.
Do Not Automate Blindly
Live gamma, 0DTE options flow, intraday futures levels, and stale web pages require same-session validation before execution.
Best Practice
Use the dashboard for regime and setup planning, then confirm with live market data before any leveraged futures decision.
